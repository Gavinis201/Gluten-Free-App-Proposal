Project Overview

A focused first version for gluten-free park dining discovery

This proposal outlines a custom website designed to help Disneyland guests locate gluten-free and celiac-friendly dining options. The first version will include interactive park maps, restaurant detail pages, curated gluten-free menu information, user accounts, reviews, photo uploads, community updates, paid access, and private owner tools for managing the content that appears on the site.

Included Features

What is included in the first version

  • User sign-in and account creation: guests can create accounts, securely log in, verify their account by email, and reset forgotten passwords.
  • Role-based permissions: different access levels will be set up for admins, approved contributors, paid users, and general users so each person can only access the features intended for them.
  • Contributor approval questions: new users can answer onboarding questions provided by Anna, and those answers will be sent to the admin area for review before the user is approved to review restaurants or upload images.
  • Subscription or one-time payment access: users can sign up for a subscription or make a one-time payment through Stripe or PayPal, depending on the final access model selected.
  • Interactive park maps: each park can have its own map, restaurant pins, and gluten-free dining locations.
  • Restaurant and menu pages: guests can view descriptions, hours, photos, prices, gluten-free status, and notes about possible gluten exposure.
  • Search and filters: guests can search by park, restaurant, or menu item and filter by meal type, dedicated fryer, mobile ordering, and location.
  • Guest reviews and photos: signed-in users can leave ratings, write reviews, upload photos, and share whether they have celiac disease, gluten sensitivity, or a gluten-free preference.
  • Email notifications: setup for account verification emails, forgotten password emails, and the sending email address used for those notifications.
  • Community updates: guests can suggest menu changes, corrections, or temporary availability updates for owner approval before anything appears publicly.
  • Owner management tools: the site owner can manage the content and user submissions that appear on the website.
  • Mobile-friendly design: the website will work across phones, tablets, and desktop computers.
  • Artwork and visual assets: custom artwork, icons, and supporting visuals needed for the first version are included.
  • Launch setup: domain setup, GitHub organization, hosting setup, and making the website live are included.

Private Owner Tools

Admin Content Management

  • Park Management: create, edit, delete, reorder, and upload park maps.
  • Map Management: place restaurant pins visually on the uploaded map image.
  • Restaurant Management: create, edit, archive, and manage restaurant information.
  • Menu Management: create, edit, archive, and organize menu items by restaurant and meal type.
  • Review Management: view, approve, remove, or moderate reviews and uploaded photos.
  • Contributor Approval Management: review answers to Anna's onboarding questions and approve users before they can post reviews or upload images.
  • Permissions Management: manage which users can access admin tools, paid content, review posting, image uploads, and contributor-only features.
  • Community Update Management: approve or reject suggested changes before they appear publicly.
  • Account Email Settings: configure the email address used for account verification and forgotten password messages.

These costs will usually start near the lower end of the ranges below and increase as more users use the app, more photos are uploaded, more emails are sent, and more data is stored.
